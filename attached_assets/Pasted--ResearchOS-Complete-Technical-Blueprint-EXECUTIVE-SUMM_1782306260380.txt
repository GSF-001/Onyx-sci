# ResearchOS: Complete Technical Blueprint

---

## EXECUTIVE SUMMARY

ResearchOS is a unified research intelligence platform that combines literature discovery, knowledge management, research gap identification, and AI-powered research assistance. By integrating capabilities from Sci-Hub, Google Scholar, Semantic Scholar, ResearchRabbit, Connected Papers, Mendeley, Notion, Obsidian, and ChatGPT into a single cohesive system, ResearchOS reduces academic research effort by 80% through automation, intelligent synthesis, and predictive insights.

**Core Value Proposition:**
- Single unified research workspace (vs. 8-10 fragmented tools)
- AI-powered research gap discovery (eliminate manual literature review)
- Real-time novelty scoring (identify high-impact research opportunities)
- Collaborative knowledge graphs (share insights across teams)
- Compliance-aware paper access (legal, auditable, transparent)

# PART 1: PRODUCT ARCHITECTURE

## 1.1 Product Vision

### Strategic Positioning
ResearchOS is the operating system for modern research—a single, intelligent platform where researchers spend 80% less time finding papers, understanding context, identifying gaps, and synthesizing knowledge.

**Time Allocation Transformation:**
- Before: 40% finding papers → 5% (AI-powered discovery)
- Before: 30% understanding context → 10% (AI summarization + graph)
- Before: 20% identifying gaps → 5% (gap engine)
- Before: 10% synthesis → 80% (deep creative research)

### Market Opportunity
- **TAM:** 50M+ researchers globally (academia, pharma, biotech, enterprise R&D)
- **SAM:** 5M+ researchers in high-value sectors (biotech, pharma, AI/ML, materials science)
- **SOM Year 1:** 10K researchers (universities + biotech firms)

### Core Thesis
Academic research is fundamentally broken as a workflow. Researchers use 8-10 disconnected tools, each with different data, different UIs, and different search paradigms. This fragmentation creates:
- 40% time waste on tool-switching and duplicate effort
- Information gaps and blind spots
- Inability to identify emerging research trends
- Siloed knowledge within institutions

ResearchOS solves this by creating a **single, intelligent research nervous system**.

**Current State (Without ResearchOS):**
```
Week 1-2:   Search Google Scholar, Pubmed, arXiv manually
            Save 500+ papers to various tools
            Manually sort and tag
            
Week 2-4:   Read abstracts, PDFs, take notes in Notion
            Create manual citation network in Excel
            Track gaps in understanding
            
Week 4-8:   Conduct secondary searches for gaps

            Synthesize findings in Word doc
            
Week 8+:    Missing key papers, backtrack, re-search
            Find contradictions in manual notes
            Redo synthesis

```

```
            ResearchOS returns 5000+ papers with automatic ranking
            AI generates landscape analysis: key subtopics, researcher network,

            AI generates follow-up searches: "Who's working on this gap?"

Week 2:     AI generates complete literature review draft with 
            citations, gaps analysis, future directions
            Researcher adds original insights, AI suggests relevant sections

            AI identifies which gaps align with advisor's expertise + 
            available resources
```

1. Natural language research question → Comprehensive landscape
2. Paper collection → AI-assisted categorization
3. Gap identification → Prioritized research directions
4. Synthesis → Draft literature review with AI assistance
5. Collaboration → Share findings with advisor/lab

## 1.4 Core Workflows

### Workflow A: Paper Discovery & Triage

**Input:** Research question / keywords / researcher profile
**Output:** Ranked, categorized, contextualized paper list

**Steps:**
1. User enters research question (natural language or keywords)
2. System queries multiple sources (arXiv, PubMed, bioRxiv, proprietary indices)
3. Deduplication engine merges results across sources
4. Ranking engine scores by: relevance, novelty, impact, recency, network position
5. AI generates category labels (automatically groups into themes)
6. System presents: ranked list, landscape visualization, gap analysis

**Success Metrics:**
- Time to find first "must-read" paper: <5 min
- Comprehensive coverage: >95% of relevant papers included
- False positive rate: <5% (irrelevant papers in top 10)

---

### Workflow B: Paper Understanding & Synthesis

**Input:** Single paper or collection of papers
**Output:** Structured understanding, cross-paper synthesis, gap analysis

**Steps:**
1. User selects paper(s)
2. System extracts: abstract, figures, tables, methodology, results, claims, limitations
3. AI generates: structured summary, key claims, methodology overview, limitations, gaps
4. Cross-paper mode: AI identifies: contradictions, complementary findings, gaps between papers
5. Citation network: Shows how selected papers connect to the field
6. Concept mapping: Extracts key concepts and visualizes relationships

**Success Metrics:**
- Summary accuracy: >90% of key claims captured
- Contradiction detection: >80% sensitivity, <10% false positive
- Time to understand paper: 5 min vs. 30 min manual

---

### Workflow C: Research Gap Identification

**Input:** Research landscape (collection of papers or research area)
**Output:** Prioritized list of research gaps with impact/difficulty scoring

**Steps:**
1. System ingests papers/patents in domain
2. Gap engine analyzes: what questions are NOT being asked?
3. Scoring: impact potential, difficulty, resource requirements, team capability alignment
4. Visualization: Gap landscape showing potential, competitiveness, resource needs
5. Trend analysis: Which gaps are emerging vs. saturating?
6. Opportunity scoring: Which gaps have best risk/reward profile?

**Success Metrics:**
- Gap identification: >80% of expert-identified gaps detected
- Quality of rankings: >90% correlation with researcher assessment
- Actionability: >70% of gaps are feasible research projects

---

### Workflow D: Continuous Monitoring & Alerting

**Input:** User-defined research areas, competitor lists, innovation keywords
**Output:** High-signal alerts, weekly briefing, trend analysis

**Steps:**
1. System monitors new papers, patents, preprints daily
2. For each new content: scoring for relevance, novelty, competitive interest
3. Alert generation: Surfaces only top 1% of new content (high signal)
4. Weekly digest: Aggregates 7 days of activity with synthesis
5. Trend analysis: Emerging topics, techniques, teams
6. Competitive alerts: Competitor activity tracking

**Success Metrics:**
- Alert signal-to-noise ratio: >10:1 (>90% of alerts are actionable)
- Coverage: >99% of relevant new papers captured
- User satisfaction: >80% of users find alerts valuable

---

### Workflow E: Collaborative Research & Team Sharing

**Input:** Research findings, papers, gaps, insights from individual researchers
**Output:** Team-shared knowledge graph, collaborative workspace

**Steps:**
1. Researcher creates discovery/collection/insight
2. Shares with team/lab
3. Team members annotate, add insights, cross-reference
4. System detects connections across team members' work
5. Collaborative notes/synthesis
6. Team briefing generation from collaborative workspace

**Success Metrics:**
- Adoption across 3+ team members: >70% of teams
- Knowledge reuse: >50% of new discoveries reference existing team knowledge
- Time to team alignment: <1 day vs. <1 week manual

---

### Workflow F: Strategic Research Planning

**Input:** Research portfolio, strategic priorities, resource constraints
**Output:** Recommended research directions, resource allocation, competitive analysis

**Steps:**
1. Program officer inputs: current projects, team capabilities, funding constraints
2. System maps against global research landscape
3. Gap analysis: where are portfolio gaps relative to landscape?
4. Opportunity scoring: which gaps have best funding potential?
5. Competitive analysis: who else is funding these areas?
6. Strategic recommendations: where to place next round of funding?
7. Risk assessment: which areas are becoming saturated?

**Success Metrics:**
- Strategic gap identification: >85% of expert-identified portfolio gaps
- Opportunity ranking: >80% correlation with strategic value
- Portfolio optimization: identifying 3-5 high-impact funding opportunities

## 1.5 Competitive Advantages

### 1. **Unified Research Nervous System**
- **What:** Single platform replacing 8-10 fragmented tools
- **Why it matters:** 40% time savings from eliminating tool-switching
- **Hard to replicate:** Requires deep integrations across literature sources, knowledge management, AI, and collaboration
- **Barrier:** Network effects (more tools integrated = more valuable; creates high switching cost)

### 2. **AI-Powered Research Gap Discovery Engine**
- **What:** Automated identification of under-researched areas with high impact potential
- **Why it matters:** Researchers spend 20% of time manually identifying gaps; engine reduces to 5%
- **Hard to replicate:** Requires large research corpus + sophisticated semantic understanding + novelty scoring algorithms
- **Barrier:** Data moat (more papers ingested = better gap detection)

### 3. **Novelty Scoring & Trend Velocity Analysis**
- **What:** Real-time assessment of research novelty, saturation, and trend direction
- **Why it matters:** Helps researchers and investors identify inflection points before they're obvious
- **Hard to replicate:** Requires continuous monitoring of global research corpus + advanced NLP + trend analysis algorithms
- **Barrier:** Time-series data moat (requires 3+ years of trend data)

### 4. **Semantic Research Knowledge Graph**
- **What:** Multi-modal knowledge graph connecting papers, researchers, methods, datasets, results, and concepts
- **Why it matters:** Enables cross-disciplinary insight discovery and connection-based navigation
- **Hard to replicate:** Requires dense, high-quality entity extraction and relationship modeling at scale
- **Barrier:** Data quality moat (manual curation + ML training)

### 5. **Compliance-Aware Paper Access**
- **What:** Legal, transparent, auditable access to research papers
- **Why it matters:** Addresses sci-hub users' needs without legal risk
- **Hard to replicate:** Requires licensing agreements, legal expertise, institutional relationships
- **Barrier:** Strategic partnerships with publishers and institutions

### 6. **Research-Optimized AI Copilot**
- **What:** Context-aware AI assistant specifically trained for research workflows
- **Why it matters:** General LLMs hallucinate; research copilot has domain-specific training and fact-checking
- **Hard to replicate:** Requires domain-specific training data, research-specific evaluation, continuous refinement
- **Barrier:** Domain expertise moat + proprietary training data

### 7. **Institutional Research Portfolio Management**
- **What:** Tools for research leaders to manage portfolios at scale with AI-powered strategic analysis
- **Why it matters:** Enables data-driven strategic research planning
- **Hard to replicate:** Requires deep understanding of research management workflows + AI capabilities
- **Barrier:** Enterprise sales complexity and domain expertise

### 8. **Real-Time Competitive Intelligence**
- **What:** Automated monitoring of competitor research, patent filings, team formations
- **Why it matters:** Researchers and VCs can spot market opportunities before competitors
- **Hard to replicate:** Requires continuous monitoring, integration with patent databases, sophisticated entity recognition
- **Barrier:** Data integration complexity + real-time processing at scale

## 2.2 Service Architecture

### Core Services (14 Total)

#### **1. Authentication & Authorization Service**

**Responsibilities:**
- User registration, login, password management
- MFA (TOTP, SMS, email-based)
- OAuth integration (Google, GitHub, ORCID)
- SSO integration (Okta, Azure AD)
- Session management
- Permission/role-based access control (RBAC)
- API key management for programmatic access
- Audit logging for all auth events

**Dependencies:** PostgreSQL, Redis, Secrets Manager
**External:** OAuth providers, SSO providers, email service

**Key Entities:**
- Users, Roles, Permissions
- Sessions, API Keys
- OAuth connections, SSO configurations
- Audit logs

---

#### **2. Discovery Service**

**Responsibilities:**
- Search paper indices (full-text, semantic, hybrid)
- Query multiple paper sources (arXiv, PubMed, bioRxiv, etc.)
- Result deduplication
- Initial ranking (relevance, recency, citations)
- Result pagination and caching
- Suggest related searches / autocomplete
- Save searches

**Dependencies:** Qdrant (vector search), PostgreSQL (metadata), Redis (cache)
**External:** arXiv API, PubMed API, bioRxiv API, Elasticsearch

**Key Entities:**
- SearchQueries, SearchResults, SavedSearches
- PaperMetadata (indexed)

**Performance Targets:**
- <500ms for full-text search across 100M papers
- <1000ms for semantic search (vector similarity)
- <200ms for hybrid search (combined ranking)
- <100 QPS capacity per instance (scale horizontally)

---

#### **3. Paper Service**

**Responsibilities:**
- Paper ingestion (metadata, PDFs, extracted content)
- PDF storage and retrieval
- PDF text extraction
- Metadata normalization and deduplication
- Figure and table extraction
- Citation extraction from PDFs
- Paper versioning (preprint → published → corrected)
- Access rights management (legal/compliance)

**Dependencies:** PostgreSQL, S3, PDF extraction library, NLP
**External:** arXiv, PubMed, bioRxiv, CrossRef API

**Key Entities:**
- Papers, Versions, Metadata
- PDFs (S3 references)
- Extracted content (figures, tables, citations, text)
- Access logs

**Processing:**
- Ingest 10,000 papers/day initially (scales to 100,000+/day)
- Extract key content in <10 seconds per paper
- Deduplicate across sources

---

#### **4. Knowledge Graph Service**

**Responsibilities:**
- Build and maintain semantic knowledge graph
- Entity extraction (concepts, methods, datasets, people, organizations)
- Relationship inference (conceptual, methodological, institutional)
- Graph queries (shortest path, influence analysis, community detection)
- Graph visualization data generation
- Knowledge graph updates (incremental ingestion)
- GraphQL interface for graph queries

**Dependencies:** Neo4j or ClickHouse (graph DB), Python (graph algorithms)
**External:** None (internal data only)

**Key Entities:**
- Concepts, Methods, Datasets, Researchers, Organizations
- Papers (as nodes)
- Relationships: cites, uses, develops, studies, contradicts, extends, etc.
- Graph snapshots (for versioning/history)

**Graph Size:**
- 100M papers → ~1B entities (concepts, methods, datasets, researchers, orgs)
- ~5B relationships
- Requires 500GB+ storage at full scale

---

#### **5. Novelty Engine**

**Responsibilities:**
- Calculate novelty scores for papers (0-100)
- Identify emerging topics vs. saturating topics
- Trend velocity analysis (acceleration of publication rate)
- Topic saturation modeling
- Competitive intensity scoring
- Predict future research trends
- Generate novelty-based alerts

**Dependencies:** PostgreSQL, time-series data, NLP embeddings
**Algorithms:** Statistical trend analysis, topic modeling, semantic similarity

**Key Entities:**
- NoveltyScores (per paper, recalculated over time)
- TopicTrends (publication rate, novelty trend over time)
- CompetitiveIntensity (paper count by topic over time)
- TrendPredictions

**Scoring Model:**

```
NOVELTY_SCORE = (
  0.3 * CONCEPTUAL_NOVELTY +       // New concepts introduced?
  0.2 * METHODOLOGICAL_NOVELTY +   // New methods?
  0.2 * EMPIRICAL_NOVELTY +        // New experimental results?
  0.15 * SEMANTIC_DISTANCE +       // How far from existing work?
  0.15 * TREND_VELOCITY             // Is this topic accelerating?
) * (1 - SATURATION_DISCOUNT)
```

Where:
- SATURATION_DISCOUNT = min(0.8, (papers_in_topic_last_year / median_historical_rate) ^ 0.5)
- Accounts for field maturity

#### **6. Gap Analysis Engine**

**Responsibilities:**
- Identify research gaps at multiple levels:
  - Topic-level gaps (understudied combinations of concepts)
  - Methodological gaps (questions not approached with certain methods)
  - Empirical gaps (predictions untested)
  - Cross-disciplinary gaps (insights from field A not applied to field B)
- Gap scoring (impact potential, difficulty, resource requirements)
- Opportunity scoring for research teams
- Gap lifecycle tracking (emerging vs. saturating)
- Gap recommendations for research planning

**Dependencies:** Knowledge graph, novelty engine, statistical analysis
**Algorithms:** Set theory (uncovered concept combinations), semantic analysis

**Key Entities:**
- ResearchGaps, GapScores
- GapOpportunities, GapRecommendations
- GapLifecycle tracking

**Gap Detection Algorithms:**

```
1. CONCEPT_COMBINATION_GAPS:
   For each pair of concepts (C1, C2) in knowledge graph:
     existing_papers = papers mentioning both C1 and C2
     if existing_papers < threshold:
       gap = CONCEPT_GAP(C1, C2)
       score = IMPACT_SCORE(C1, C2, related_fields)

2. METHODOLOGICAL_GAPS:
   For each question Q and method M:
     studied = papers studying Q with method M
     if studied < threshold:
       gap = METHOD_GAP(Q, M)
       score = POTENTIAL_IMPACT(Q, M)

3. CROSS_DISCIPLINARY_GAPS:
   For each technique T in field F1:
     relevant_to = fields where T could apply
     if used_in = fields where T is already used:
       unexplored = relevant_to - used_in
       for field F2 in unexplored:
         gap = XDISCIPLINARY_GAP(T, F2)
         score = MARKET_POTENTIAL(T, F2)
```

### 2.5.1 Research Copilot Pipeline

```
User Query (Research Question)

    ┌───────────────────────────┐
```

**Key Components:**

**1. Query Understanding**
- Intent classification: summarization, comparison, methodology, gap analysis, prediction
- Entity extraction: concepts, datasets, methods, researchers, organizations
- Question clarification (if ambiguous)

**2. Multi-Stage Retrieval**

*Stage 1: Broad Search (Recall-Optimized)*
- Semantic search: Query embedding → vector similarity search in Qdrant
- Keyword search: BM25 across full text and metadata
- Citation network search: Find papers highly cited by relevant papers
- Knowledge graph search: Find papers connected via concepts

*Stage 2: Ranking & Filtering (Precision-Optimized)*
```
RANKING_SCORE = (
  0.3 * SEMANTIC_SIMILARITY +
  0.2 * CITATION_NETWORK_SCORE +
  0.2 * AUTHORITY_SCORE +
  0.15 * RECENCY_SCORE +
  0.15 * CONFLICT_RESOLUTION_SCORE
)

where:
- AUTHORITY_SCORE = h_index_percentile_of_first_author
- RECENCY_SCORE = 1 / (years_since_publication ^ 0.5)
- CONFLICT_RESOLUTION = positive_if_supports_consensus, 
                        negative_if_contradicts
```

**3. Context Assembly**
- Extract: abstract, methodology, key results, limitations, future work
- Add: knowledge graph context (related concepts, methods, datasets)
- Build: structured context window (max 8000 tokens before response)

**4. LLM Generation**

*Specialized Prompts for Different Tasks:*

**Summarization Prompt:**


### 2.5.2 Novelty Scoring Model

**Inputs:**
- Paper embeddings (semantic content)
- Citation network (who cites whom)
- Metadata (publication date, field, keywords)
- Paper concepts (extracted entities)
- Historical publication trends

**Scoring Formula:**

```
NOVELTY_SCORE(paper, field) = (
  0.25 * CONCEPTUAL_NOVELTY(paper) +
  0.20 * METHODOLOGICAL_NOVELTY(paper) +
  0.15 * EMPIRICAL_NOVELTY(paper) +
  0.20 * SEMANTIC_DISTANCE(paper, field) +
  0.20 * TREND_VELOCITY(field)
) * (1 - SATURATION_PENALTY(field))

where:

CONCEPTUAL_NOVELTY = 
  1 - max_cosine_similarity_to_previous_papers_concepts

METHODOLOGICAL_NOVELTY =
  1 - max_similarity_to_previous_papers_methodologies

EMPIRICAL_NOVELTY =
  avg_distance_to_nearest_k_papers +
  novelty_of_datasets_used +
  novelty_of_experimental_setup

SEMANTIC_DISTANCE(paper, field) =
  1 - (avg_similarity_to_field_papers / max_possible_similarity)

TREND_VELOCITY(field) = 
  ln(papers_last_year / papers_previous_year)

SATURATION_PENALTY(field) =
  min(1.0, papers_in_field_last_year / 
          (historical_median_rate * 2.5))
```

**Calibration:**
- Trained on citation count growth over 2+ years (papers with >10X citation growth = highly novel)
- Validated against expert researcher novelty judgments
- Field-specific calibration (different for AI vs. biology)

---

### 2.5.3 Embedding Model

**Strategy:** Proprietary fine-tuned embedding model
- Base: OpenAI ada-002 or similar (1536 dimensions)
- Fine-tuning: Research paper citation pairs (papers frequently cited together are semantically similar)
- Dataset: 50M+ citation pairs from S2ORC

**Update Frequency:**
- Continuously retrained on new citations
- Major update every quarter
- Full reindexing every 6 months

**Storage:**
- Qdrant vector database
- 100M papers × 1536 dims × 4 bytes = ~614GB
- Replicated 3x for redundancy and query distribution

---

### 2.5.4 Ranking Model

**Learning-to-Rank (LTR) Model:**
- Gradient Boosted Trees (LightGBM or XGBoost)
- Features: semantic similarity, citation count, author h-index, recency, field match, etc.
- Labels: click-through rate, save rate, time spent on paper, user feedback
- Retraining: Weekly on new user interaction data

**Offline Training Pipeline:**
1. Sample 100K user search sessions
2. Extract features for each paper in result set
3. Label by user behavior (click = positive, skip = negative)
4. Train LTR model
5. Validation on holdout test set
6. Deploy if performance improves

## 2.6 Search Architecture

### 2.6.1 Indexing Strategy

**Paper Content Indexed:**

3. Keywords: extracted key terms from NLP

```
    ↓
Ready for search queries
```

**Indexing Performance:**

- Latency SLA: indexed and queryable within 5 minutes

### 2.6.2 Hybrid Search Ranking

**Search Query Flow:**

```
User Query: "deep learning drug discovery"
    ↓
[Query Understanding]
    ├─ Tokenization
    ├─ Query expansion (synonyms)
    ├─ Field detection (title, abstract, full-text)
    └─ Intent classification (search type)
    ↓
[Parallel Search Execution]
    ├─ Keyword Search (Elasticsearch BM25)
    │   └─ Top 1000 results by relevance
    ├─ Semantic Search (Qdrant vector search)
    │   └─ Top 1000 results by cosine similarity
    ├─ Citation Network Search
    │   └─ Find highly-cited papers in result set
    └─ Knowledge Graph Search
        └─ Find papers with related concepts
    ↓
[Result Merging & Ranking]
    For each unique paper in combined results:
    
    score = (
      0.35 * normalized_keyword_score +
      0.35 * normalized_semantic_score +
      0.15 * normalized_citation_score +
      0.10 * normalized_recency_score +
      0.05 * normalized_authority_score
    )
    
    Normalize each score to [0, 1] range
    ↓
[Top Results]
    Return top 50 papers, ranked by composite score
    ↓
[Response]
    Display with snippets, highlights, and metadata
```

**Algorithm Details:**

**1. Keyword Search (Elasticsearch):**
```
BM25(query, document) = ∑ IDF(qi) * (f(qi, D) * (k1 + 1)) / 
                                    (f(qi, D) + k1 * (1 - b + b * |D| / avgdl))

Parameters:
- k1 = 1.2 (term frequency saturation)
- b = 0.75 (field length normalization)
- avgdl = average document length

Field-specific weights:
- Title: 5x
- Abstract: 2x
- Full-text: 1x
- Keywords: 3x
```

**2. Semantic Search (Qdrant):**
```
cosine_similarity(query_embedding, paper_embedding) = 
  (query_vec · paper_vec) / (|query_vec| * |paper_vec|)

With approximate nearest neighbor search for scale
```

**3. Citation Network Score:**
```
citation_score(paper) = 
  ln(1 + citation_count) +
  0.5 * recency_boost(publication_year)

where recency_boost increases score for recent highly-cited papers
```

**4. Authority Score:**
```
authority_score(paper) = 
  mean(h_index_percentile of first 3 authors) +
  0.3 * ln(1 + total_author_citations)
```

---

### 2.6.3 Search Performance Optimization

**Caching Strategy:**
```
Level 1: In-Memory Query Cache (Redis)
  - Cache common searches for 1 hour
  - Hit rate target: 40-50% of searches
  - Storage: 10GB

Level 2: Elasticsearch Query Cache
  - Automatic caching by Elasticsearch
  - LRU eviction

Level 3: Vector Cache (Qdrant)
  - Cache popular embeddings
  - Hit rate: 60-70%
```

**Query Optimization:**
- Precompute popular searches offline
- Query expansion for common synonyms
- Autocomplete with prefix search
- Suggestion ranking

**Infrastructure Scaling:**
```
Search QPS (Queries Per Second) vs. Infrastructure:

1,000 QPS:    Single Elasticsearch node (16GB RAM)
10,000 QPS:   3-node Elasticsearch cluster
100,000 QPS:  10-node Elasticsearch cluster + caching layer
1,000,000 QPS: Multi-region Elasticsearch + Memcached

Note: Assumes 80/20 distribution (80% cache hits, 20% new queries)
```

## 4.5 Query Patterns & Optimization

### High-Traffic Query Patterns:

**1. Search Query (Most Common)**
```sql
SELECT p.id, p.title, p.abstract, p.publication_date, 
       p.citation_count, p.novelty_score,
       array_agg(pa.author_name) AS authors
FROM papers p
LEFT JOIN paper_authors pa ON p.id = pa.paper_id
WHERE p.field = $1 AND p.subfield = $2
  AND p.publication_date > NOW() - INTERVAL '5 years'
  AND p.novelty_score > $3
ORDER BY p.novelty_score DESC, p.citation_count DESC
LIMIT 50;

-- Optimization:
-- - Index: (field, subfield, novelty_score DESC)
-- - Materialize authors in paper table or denormalize
-- - Cache results for 1 hour
```

**2. Paper Citation Network**
```sql
SELECT cited_paper_id, COUNT(*) as citation_count
FROM paper_citations
WHERE citing_paper_id = $1
GROUP BY cited_paper_id
ORDER BY citation_count DESC
LIMIT 20;

-- Optimization:
-- - Precompute and cache (update daily)
-- - Store directly in papers table as a JSONB field
-- - Update in batch job rather than real-time
```

**3. User Research Collection**
```sql
SELECT p.id, p.title, cp.added_by_user_id, cp.user_notes
FROM collection_papers cp
JOIN papers p ON cp.paper_id = p.id
WHERE cp.collection_id = $1
ORDER BY cp.date_added DESC;

-- Optimization:
-- - Composite index: (collection_id, date_added DESC)
-- - Cache collection contents for 1 day
```

**4. Trending Topics**
```sql
SELECT topic, SUM(paper_count) as total_papers,
       AVG(trending_velocity) as avg_velocity
FROM topic_trends
WHERE metric_date >= NOW() - INTERVAL '30 days'
GROUP BY topic
ORDER BY avg_velocity DESC
LIMIT 20;

-- Optimization:
-- - Index: (metric_date DESC)
-- - Precompute this query daily and cache results
```

# PART 5: SEARCH ARCHITECTURE

## 5.1 Paper Indexing

### Data Normalization Pipeline:

```
Raw Paper (PDF + Metadata)
    ↓
[Extraction Service]
    ├─ Extract text from PDF
    ├─ Extract metadata (title, authors, abstract)
    ├─ Extract citations
    ├─ Extract figures and tables
    ├─ Extract concepts (NER)
    └─ Extract methods (domain-specific)
    ↓
[Normalization Service]
    ├─ Clean text (remove special chars, normalize unicode)
    ├─ Normalize author names (surname standardization)
    ├─ Normalize concepts (resolve synonyms, abbreviations)
    ├─ Resolve citations (link to known papers)
    └─ Deduplicate paper versions
    ↓
[Enrichment Service]
    ├─ Generate embeddings
    ├─ Extract key findings
    ├─ Identify contradictions with existing papers
    ├─ Compute novelty score
    ├─ Assign field/subfield
    └─ Map to knowledge graph concepts
    ↓
[Indexing Service]
    ├─ Index full-text (Elasticsearch)
    ├─ Index embeddings (Qdrant)
    ├─ Store metadata (PostgreSQL)
    ├─ Update knowledge graph (Neo4j)
    └─ Update cache (Redis)
    ↓
Ready for Search
```

### Metadata Normalization:

```
1. Author Name Normalization:
   Input: ["John Smith", "Smith, John", "John Q. Smith"]
   Output: "Smith, John"
   
2. Concept Normalization:
   Input: ["Machine Learning", "ML", "Deep Learning", "Neural Networks"]
   Output: Standardized to concept IDs in knowledge graph
   
3. Institution Normalization:
   Input: ["MIT", "Mass Inst Tech", "Massachusetts Institute of Technology"]
   Output: map to known organization ID
   
4. Publication Date Normalization:
   Input: Various formats
   Output: ISO 8601 format (YYYY-MM-DD)
   
5. Citation Format Normalization:
   Input: Various citation formats
   Output: Structured format with ID references
```

## 5.2 Search Ranking Algorithm

### Multi-Stage Ranking Pipeline:

```
Query: "deep learning drug discovery"
    ↓
[Stage 1: Recall Phase - Get Top 1000]
    ├─ BM25 keyword search: Top 500 by relevance
    ├─ Vector semantic search: Top 500 by similarity
    └─ Graph-based search: Top 200 by connection strength
    ├─ Merge & deduplicate: ~1000 unique papers
    ↓
[Stage 2: Ranking Phase - Score & Rank]
    For each paper:
        score = (
          0.35 * norm(bm25_score) +         # Keyword relevance
          0.35 * norm(semantic_score) +     # Semantic similarity
          0.10 * norm(citation_score) +     # Citation network
          0.10 * norm(authority_score) +    # Author authority
          0.07 * norm(recency_score) +      # Publication freshness
          0.03 * norm(field_match_score)    # Field alignment
        )
    ↓
[Stage 3: Personalization Phase]
    If user_id provided:
        user_profile = load_user_interests(user_id)
        for each paper:
            personalization_boost = 0.1 if paper matches user_interests
            score = score * (1 + personalization_boost)
    ↓
[Stage 4: Result Rendering]
    Return top 50 papers with:
    - Ranking explanation (which signals were most important)
    - Snippet highlighting (keywords in context)
    - Related papers (from citation network)
    - Confidence score (how confident are we in this ranking?)
```

### Scoring Components:

**1. BM25 Keyword Relevance**
```
BM25(q, d) = Σᵢ IDF(qᵢ) * [f(qᵢ, d) * (k1 + 1)] / [f(qᵢ, d) + k1 * (1 - b + b * |d|/avgdl)]

where:
- f(qᵢ, d) = frequency of term qᵢ in document d
- |d| = length of document d
- avgdl = average document length
- k1 = 1.2 (controls term frequency saturation)
- b = 0.75 (controls field length normalization)
- IDF = inverse document frequency

Field-specific term weights:
- Title: 5x
- Abstract: 2x
- Keywords: 3x
- Full-text: 1x

Example: Paper with query term in title = 5x multiplier
```

**2. Semantic Similarity**
```
similarity = cos(embedding(query), embedding(document))

where:
- embeddings are 1536-dim vectors from fine-tuned model
- embeddings trained on citation co-occurrence
- cos(a, b) = (a · b) / (|a| * |b|)
- Range: [-1, 1], rescaled to [0, 1]
```

**3. Citation Network Score**
```
citation_score = (
  0.6 * ln(1 + citation_count) +
  0.2 * recency_boost +
  0.2 * h_index_normalized
)

where:
- recency_boost = 1 / (1 + exp(-2 * (years_old - 5)))
  (boost for papers 3-7 years old)
- h_index_normalized = first_author_h_index / 100 (capped)
```

**4. Authority Score**
```
authority_score = (
  mean(h_index_percentile of first 3 authors) * 0.7 +
  ln(1 + total_author_citations) / 5 * 0.3
)

where:
- h_index_percentile computed relative to field
- Rewards papers from well-known researchers
```

**5. Recency Score**
```
recency_score = 1 - (1 / (1 + exp((years_old - 3) / 2)))

where:
- years_old = years since publication
- Sigmoid curve: penalty for very old papers
- Peak score for papers 2-4 years old
```

**6. Personalization**
```
personalization_boost = (
  0.3 * concept_overlap +
  0.3 * method_overlap +
  0.2 * author_overlap +
  0.2 * citation_overlap
)

where:
- concept_overlap = Jaccard(paper_concepts, user_interests)
- method_overlap = similar for methods
- author_overlap = user has worked with/cited these authors
- citation_overlap = paper cites works user has saved
```

## 5.3 Hybrid Search Implementation

### Architecture:

```
User Query
    ↓
[Query Parsing & Expansion]
    ├─ Tokenize query
    ├─ Expand synonyms (use thesaurus/wordnet)
    ├─ Extract entities (concepts, methods, organisms)
    ├─ Detect query intent (search type)
    └─ Generate alternative phrasings
    ↓
[Parallel Search Execution]
    │
    ├─ Keyword Search (Elasticsearch)
    │   ├─ BM25 search with field boosts
    │   ├─ Return top 500 results with scores
    │   └─ Latency: ~100ms
    │
    ├─ Semantic Search (Qdrant)
    │   ├─ Generate query embedding
    │   ├─ HNSW approximate similarity search
    │   ├─ Return top 500 results with scores
    │   └─ Latency: ~50ms
    │
    ├─ Citation Network Search
    │   ├─ Find papers citing/cited by top keyword results
    │   ├─ Return top 200 papers
    │   └─ Latency: ~100ms
    │
    └─ Knowledge Graph Search
        ├─ Find papers with related concepts
        ├─ Traverse concept relationships
        ├─ Return top 200 papers
        └─ Latency: ~150ms
        
    Total parallel latency: ~200ms (max of all)
    ↓
[Result Merging]
    Combine all result sets, remove duplicates
    ↓
[Ranking & Scoring]
    For each unique paper:
        composite_score = (
          0.35 * norm(keyword_score) +
          0.35 * norm(semantic_score) +
          0.15 * norm(citation_network_score) +
          0.10 * norm(kg_score) +
          0.05 * norm(authority_score)
        )
    ↓
[Personalization]
    If user signed in:
        Apply user-specific boosts based on interests
    ↓
[Top Results]
    Return top 50 ranked papers
    ↓
[Response]
    Return to user with:
    - Ranked list
    - Result snippets with highlighting
    - Related papers
    - "Why this paper?" explanation
```
